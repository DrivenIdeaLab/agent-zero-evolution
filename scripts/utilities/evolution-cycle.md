# Evolution Cycle Routine

_Trigger by saying: **run evolution cycle**_

## Steps
1. **Check for Changes Since Last Cycle**
   - List new/modified files (configs, skills, hooks, prompts, docs)
   - Summarize new/modified memories
2. **Review Evolution Backlog**
   - List all pending (not DONE) items from evolution-backlog.md
3. **Health Check**
   - Confirm presence and integrity of all key components (configs, registries, skills, hooks, prompts, knowledge)
   - Flag missing or inconsistent files
4. **Propose New Backlog Items**
   - Analyze recent usage, errors, or gaps; suggest improvements
5. **Recommend Top 3 Actions**
   - Prioritize based on impact, effort, risk, and urgency
6. **Summarize in Status Report**
   - Brief, actionable summary for the operator

_This routine is designed to be run periodically or on demand._
